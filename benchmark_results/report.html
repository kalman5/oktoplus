4. Average Parallelism Scaling

Average ops/sec across all commands vs concurrent clients

Surprise: Oktoplus does NOT outscale Redis despite per-key locking. Redis peaks at ~99K ops/sec (100 clients), Oktoplus plateaus at ~68-72K. The gap is worst at 50-100 clients (0.69-0.74x). Likely causes: outer map mutex contention, thread-per-connection overhead, or shared resource bottleneck.

5. Latency at High Concurrency (200 clients)

p99 Latency (ms) — 200 concurrent clients

Tail latency: LRANGE_100 p99=8.39ms (max=196ms) on Oktoplus vs p99=2.33ms (max=2.85ms) on Redis. LPUSH also shows a 59ms max spike. Other commands are comparable.

6. Recommendations

1. Implement pipelining — 10x throughput sitting on the table. Most Redis clients pipeline by default.
2. Profile contention — Run perf/flamegraph under 50-100 client load. The 0.61x ratio on POP at 50 clients points to a specific serialization point.
3. Shard the key map — Replace the single outer mutex in ContainerFunctorApplier with a sharded map (64 shards) to reduce map-level lock contention.
4. Async I/O — Replace thread-per-connection with io_context thread pool + async_accept/async_read.
