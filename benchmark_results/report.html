Surprise: Oktoplus does NOT outscale Redis despite per-key locking. Redis peaks ~80-100K ops/sec, Oktoplus plateaus at ~70-80K. Likely causes: lock contention overhead, thread-per-connection model, or shared resource bottleneck.

6. Recommendations

1. Implement pipelining — 10x throughput sitting on the table. Most Redis clients pipeline by default.
2. Profile contention — Run perf/flamegraph under 50-200 client load to find what's limiting scaling.
3. Async I/O — Replace thread-per-connection with io_context thread pool + async_accept/async_read.
4. Test with larger payloads — Current tests use 3-byte values. Real workloads may shift the picture.
